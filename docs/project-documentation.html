Chapter 01
Project Overview

1.1 What is AI Crop Dashboard?

AI Crop Dashboard is a browser-based agricultural intelligence platform that helps farmers make data-driven decisions about which crop to grow. It combines a real neural network trained on Kaggle data, live charts from a PostgreSQL database, and a modern premium UI — all running in a single web browser with no separate backend server needed.

🧠
AI Prediction
TensorFlow.js neural network predicts the best crop based on 7 field parameters with ~97% accuracy.
📊
Live Analytics
4 interactive Chart.js charts built from real Supabase database records.
🌦
Weather Alerts
6 regional weather alerts with detailed modal popups and farmer advisories.
🔐
Secure Auth
Supabase email + password authentication with protected routes.
📋
Data Management
Farmers can submit and view field data records stored in Supabase PostgreSQL.
🎨
Premium UI
Dark mode, glassmorphism, micro-animations, gradient themes — zero frameworks.

4.3 Training Process — Step by Step

  • 1
    Page Load → TF.js CDN loadedTensorFlow.js 4.20 is loaded from jsDelivr CDN before analytics.js runs.
  • 2
    fetch('Crop_recommendation.csv')The real Kaggle CSV is fetched via HTTP from the project root. All 2200 rows are parsed into feature arrays [N,P,K,Temp,Humid,pH,Rain] and one-hot label arrays.
  • 3
    Min-Max NormalizationEach of the 7 features is scaled to [0, 1] using the column-wise min/max from the training data. The scaler parameters are saved for use during prediction.
  • 4
    Model Buildtf.sequential() creates the 7-layer network. model.compile() sets Adam optimizer (lr=0.003), categorical cross-entropy loss, accuracy metric.
  • 5
    model.fit() — 60 EpochsTraining runs for 60 epochs with batch size 16, 10% validation split, shuffle=true. Each epoch end fires onEpochEnd callback which updates the progress bar and stats in the UI.
  • 6
    Tensor DisposalAfter training, input/label tensors are disposed with .dispose() to free GPU/CPU memory.
  • 7
    Button EnabledThe "Predict Suitable Crop" button is re-enabled and the training panel updates to show ✅ accuracy, sample count, epoch count, and timestamp.
// Simplified training call (from analytics.js)
await mlModel.fit(xsTensor, ysTensor, {
  epochs:          60,
  batchSize:       16,
  validationSplit: 0.1,
  shuffle:         true,
  callbacks: {
    onEpochEnd: (epoch, logs) => {
      updateTrainingProgress(epoch + 1, 60, logs.accuracy, logs.loss);
    }
  }
});

4.4 Prediction / Inference

  • 1
    Read slider values7 values are read from the HTML range inputs using getElementById(id).value.
  • 2
    Normalize inputEach value is scaled using the saved scaler: (value - min) / (max - min). This must match how training data was normalized.
  • 3
    Create tensortf.tensor2d([normalizedInput]) creates a 1×7 tensor.
  • 4
    Forward passmodel.predict(inputTensor) runs through all 7 layers and returns a 1×22 softmax output.
  • 5
    Read probabilitiespredTensor.dataSync() reads the 22 probability values. These are sorted to find the top-4 crops.
  • 6
    Calculate suitabilityThe actual input is compared to the ideal center values for the winning crop. Percentage deviation is used to compute "Field Suitability %".
  • 7
    Render resultshowPredictionResult() injects HTML into the result container with hero card, top-4 rankings, parameter comparison, and model footer.

Backend — How Supabase Works

5.1 Supabase Overview

Supabase is an open-source Firebase alternative built on PostgreSQL. It provides a REST API, Auth service, and real-time subscriptions — accessible directly from the browser using the Supabase JavaScript SDK. No traditional backend server (Node.js, Django, etc.) is needed.

5.2 Authentication Flow

  • 1
    User enters email + passwordOn login form, user fills in credentials and clicks the Login button.
  • 2
    signInWithPassword() calledsupabase.auth.signInWithPassword({email, password}) sends a POST request to the Supabase Auth endpoint.
  • 3
    Supabase returns JWT tokenOn success, Supabase returns an access token (JWT) + refresh token. The SDK stores these in browser localStorage automatically.
  • 4
    Session persists across pagessupabase.auth.getSession() is called on every page load. If a valid token exists, user is considered logged in.
  • 5
    Logoutsupabase.auth.signOut() clears the token from localStorage and redirects to login.
// auth.js — Login
const { data, error } = await supabase.auth.signInWithPassword({
  email:    emailInput.value,
  password: passwordInput.value
});
if (error) showToast('error', error.message);
else window.location.href = 'index.html';

5.5 Row Level Security (RLS)

RLS ensures that only logged-in users can read or write to farmer_data. Every API call automatically includes the user's JWT token in the Authorization header. Supabase validates the token server-side before allowing access.

⚠️ Security Note: Never expose your service_role key in client-side code. Only use the anon (public) key — it is safe to expose and respects RLS policies.
ℹ️ supabase-config.js: Set SUPABASE_URL and SUPABASE_ANON_KEY to your own project values from the Supabase dashboard under Settings → API.

How Weather Alerts Work

Alerts are defined as a static JavaScript array WEATHER_ALERTS[] in analytics.js. Each object has: type, icon, title, description, region, time, severity, wind speed, rainfall, duration, advisory tips array, affected crops, temperature range, humidity.

  • 1
    renderWeatherAlerts()Iterates WEATHER_ALERTS array and injects alert chip HTML into #alertList container.
  • 2
    User clicks chip → openWeatherModal(idx)Creates a full modal overlay <div> dynamically and appends it to document.body.
  • 3
    Modal shows full detailHeader (icon + title + badge), description, detail rows, 4 weather stat boxes, 5-point farmer advisory list.
  • 4
    closeWeatherModal()Fades the overlay (opacity 0 over 200ms) then removes it from DOM. Also cleans up the Escape key listener.

Chapter 08
Local Server Setup

Why a Local Server is Required

⚠️ Important: You CANNOT open the app by double-clicking analytics.html (the file:// protocol). Two features require HTTP:
  1. fetch('Crop_recommendation.csv') — browsers block fetch() on file://
  2. Supabase Auth — requires an HTTP origin for token cookies

Option A — PowerShell Script (Recommended for Windows)

# Open PowerShell in the project folder and run:
.\start-server.ps1

# Then open browser at:
http://localhost:8080

Option B — Batch File

run.bat
# Double-click run.bat in Windows Explorer

Option C — Python

python -m http.server 8080
# Works on any OS with Python 3 installed

Option D — Node.js

npx serve . -p 8080
# Requires Node.js installed

Supabase Configuration

// js/supabase-config.js — Edit these 2 values
const SUPABASE_URL      = 'https://YOUR_PROJECT.supabase.co';
const SUPABASE_ANON_KEY = 'eyJ...(your anon key)...';

Get these values from: Supabase Dashboard → Settings → API

Chapter 10
Troubleshooting Guide
ProblemLikely CauseFix
Button says "Training…" and never enables TF.js CDN failed to load, or CSV fetch failed Check browser console for network errors. Ensure you're on http://localhost not file://
"CSV not found" toast appears Crop_recommendation.csv missing from project root Copy the CSV file to the same folder as analytics.html
Charts show no data Supabase not configured or no records yet Set correct URL + anon key in supabase-config.js. Add at least 1 record via the form.
"Email not confirmed" on login Supabase email confirmation is ON Supabase Dashboard → Auth → Settings → Disable "Enable email confirmations"
Sliders don't update live values JS error crashing the script Open browser DevTools (F12) → Console tab → fix any red errors shown
Prediction always shows same crop Model not trained (mlModel is null) Wait for training to complete. Green toast "Neural Network trained!" confirms readiness.
Page redirects to login immediately No valid session / JWT expired Log in again. Sessions expire after 1 hour by default in Supabase.
run.bat / start-server.ps1 doesn't work Python or PowerShell execution policy Run python -m http.server 8080 manually in terminal, or use npx serve . -p 8080
